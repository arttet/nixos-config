# 🔍 Diagnostics

When the system behaves unexpectedly, use these specialized diagnostic guides to identify the root cause. This section focuses on deep technical inspection of the various system layers.

## Diagnostic Areas

### 🚪 [GRUB & Early Boot](./grub)

Troubleshooting the very first stages of boot, including kernel parameters and emergency shell access.

### ⏱️ [Boot Time](./boot)

Analyzing systemd initialization to find and fix slow startup issues.

### 💾 [Disk Space](./disk)

Identifying large files and monitoring the health of the Btrfs filesystem.

### 🛡️ [Security & Auditing](./security)

Inspecting authentication logs, session data, and system integrity.

### 🌐 [Networking](./network)

Debugging latency, DNS resolution, and core network stack performance.
