# GUI Direction

The platform is intentionally headless before GUI work begins.

GUI will be introduced later as a separate layer. It must not be mixed into the
headless workstation baseline.

Future GUI topics include:

- seat management
- polkit
- Wayland
- portals
- audio
- display or login manager
- Hyprland
- fonts
- browser
- desktop security policy

No GUI, Hyprland, desktop portal, audio, or display manager is implemented in
the current headless foundation.
